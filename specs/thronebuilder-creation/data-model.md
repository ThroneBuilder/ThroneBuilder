# Data Model: ThroneBuilder.com Site Creation

**Phase 1 output for**: `thronebuilder-creation`
**Date**: 2026-05-26

## Entities

### Article

The primary content unit. Stored as a Markdown file in `src/content/blog/`.

**File naming convention**: `YYYY-MM-DD-slug.md`
Example: `2026-05-26-welcome-to-thronebuilder.md`

**Frontmatter schema** (`src/content/config.ts`):

```typescript
import { defineCollection, z } from 'astro:content';

const blog = defineCollection({
  type: 'content',
  schema: z.object({
    title:       z.string(),
    pubDate:     z.coerce.date(),
    description: z.string(),
    coverImage:  z.string().optional(),
    youtubeUrl:  z.string().url().optional(),
  }),
});

export const collections = { blog };
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `title` | string | Yes | Displayed as `<h1>` and page `<title>` |
| `pubDate` | date (ISO 8601) | Yes | Used for sort order and display |
| `description` | string | Yes | Used as meta description and article excerpt on homepage |
| `coverImage` | string (path or URL) | No | Relative path from `public/` (e.g., `/images/my-article.jpg`) or absolute URL. Falls back to `DEFAULT_OG_IMAGE` when absent. |
| `youtubeUrl` | string (URL) | No | Full YouTube URL (watch, share, or shorts format). Parsed to video ID at build time. Embed skipped if absent or invalid. |
| body | Markdown | Yes | Article body rendered to HTML. Not a frontmatter field — the file content below the frontmatter delimiter. |

**Validation rules**:
- `title` MUST be non-empty.
- `pubDate` MUST be a valid ISO 8601 date string (e.g., `2026-05-26`).
- `description` MUST be non-empty (used as meta description; keep under 160 chars
  for SEO best practice, though not enforced at build time).
- `youtubeUrl`, when present, MUST be a valid URL. The build will error on invalid
  URLs due to Zod `.url()` validation.

**Draft/Published**: Articles are always published once the file exists in
`src/content/blog/`. To draft an article, do not commit it to `main`. There is
no `draft` field in v1 — this can be added in a later iteration if needed.

### Site Config

Static constants committed to `src/consts.ts`. Not a database record — a
TypeScript file imported directly by layouts and pages.

```typescript
// src/consts.ts

export const SITE_TITLE       = 'ThroneBuilder';
export const SITE_DESCRIPTION = 'Strategy, history, and analysis for the discerning player.';
export const SITE_URL         = 'https://thronebuilder.com';
export const DEFAULT_OG_IMAGE = '/images/og-default.png';
```

| Constant | Purpose |
|----------|---------|
| `SITE_TITLE` | `<title>` tag fallback; RSS channel title |
| `SITE_DESCRIPTION` | Homepage meta description; RSS channel description |
| `SITE_URL` | Canonical base URL; used by sitemap + RSS integrations |
| `DEFAULT_OG_IMAGE` | Fallback `og:image` when article has no `coverImage` |

**Note**: `SITE_URL` is also set in `astro.config.mjs` as the `site` property
(required by `@astrojs/sitemap` and `@astrojs/rss`). Both values must match.

## Derived Data (Build-Time Only)

These are not stored entities — they are computed at build time from the above.

### Article List (homepage)

Computed by `src/pages/index.astro`:

```typescript
const posts = (await getCollection('blog'))
  .sort((a, b) => b.data.pubDate.getTime() - a.data.pubDate.getTime());
```

Fields used: `slug` (for URL), `title`, `pubDate`, `description`.

### RSS Feed Items

Computed by `src/pages/rss.xml.ts` using `@astrojs/rss`.
Fields used: `title`, `pubDate`, `description`, `slug` (for link).

### Sitemap

Auto-generated by `@astrojs/sitemap` from all pages discovered during the
Astro build. No manual configuration required beyond setting `site` in
`astro.config.mjs`.

### YouTube Video ID

Extracted from `youtubeUrl` frontmatter at build time in `[...slug].astro`:

```typescript
function extractYoutubeId(url: string): string | null {
  const patterns = [
    /[?&]v=([^&#]+)/,        // watch?v=
    /youtu\.be\/([^?&]+)/,   // youtu.be/
    /\/shorts\/([^?&]+)/,    // /shorts/
  ];
  for (const pattern of patterns) {
    const match = url.match(pattern);
    if (match) return match[1];
  }
  return null;
}
```

Returns `null` for unrecognized URL formats — the embed is skipped silently.

## Entity Relationships

```text
Site Config (1)
    └── provides defaults for → Article (many)
                                    ├── title
                                    ├── pubDate
                                    ├── description
                                    ├── coverImage?  → public/images/
                                    └── youtubeUrl?  → YouTube (external)
```

No relational data. No joins. No IDs beyond the filename slug.
